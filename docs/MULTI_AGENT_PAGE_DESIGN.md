## 一、产品背景与目标

### 1.1 现状问题

用户反馈当前 MultiAgentPage 存在三大能力缺口：

| 问题 | 描述 |
|------|------|
| **能力编辑受限** | 无法编辑子 Agent 的技能（Skills）、工具（Tools）、系统提示词（System Prompt） |
| **缺乏任务编排** | 无法为多 Agent 协作定义执行顺序、并行关系和条件分支 |
| **协议选择缺失** | 无法选择 Agent 间的交互协议（文件传递 / 协议通信 / 混合模式） |

### 1.2 设计目标

- **能力可配**: 每个子 Agent 的 LLM、技能、工具、提示词均可独立配置
- **编排灵活**: 支持顺序、并行、条件分支等多种执行模式
- **协议透明**: 支持 File-based / Protocol-based / Hybrid 三种交互协议
- **体验一致**: 保持与现有 MultiAgentPage 风格统一，降低学习成本

## 四、Tab 2: Agent 能力配置 🆕

### 4.1 设计目标

对选中的单个 Agent 进行深度能力配置，涵盖 LLM 底座、技能、工具、提示词四个维度。

### 4.2 子 Tab 结构

```
┌─────────────────────────────────────────────────────┐
│ [🤖 LLM 配置] [🛠️ 技能选择] [🔧 工具配置] [💬 提示词编辑] │
└─────────────────────────────────────────────────────┘
```

### 4.3 子 Tab 详细内容

---

#### 4.3.1 🤖 LLM 配置

**功能**: 选择该 Agent 使用的大模型底座及参数。

**配置项**:

| 字段 | 类型 | 说明 |
|------|------|------|
| **模型选择** | 下拉单选 | 支持的模型列表（如 GPT-4o、Claude-3.5、GLM-4 等） |
| **温度 (Temperature)** | 滑块 | 0.0 ~ 2.0，默认 0.7 |
| **最大 Token 数** | 数字输入 | 限制单次回复最大 Token 数 |
| **Top-P** | 滑块 | 0.0 ~ 1.0，默认 0.9 |
| **频率惩罚** | 滑块 | -2.0 ~ 2.0，默认 0 |
| **存在惩罚** | 滑块 | -2.0 ~ 2.0，默认 0 |
| **API Base URL** | 文本输入 | 自定义模型接口地址（可选） |
| **API Key** | 密码输入 | 自定义接口密钥（加密存储） |

**交互**:
- 切换模型时自动加载该模型的默认参数
- 修改参数后显示 "●" 未保存标识
- 提供「重置为默认」按钮

---

#### 4.3.2 🛠️ 技能选择

**功能**: 为该 Agent 绑定可用的技能（Skills）。

**配置项**:

| 字段 | 类型 | 说明 |
|------|------|------|
| **可用技能列表** | 多选列表 | 展示所有已安装的 Skill，支持多选 |
| **技能详情预览** | 折叠面板 | 选中技能后展示其描述和用途 |
| **技能执行优先级** | 拖拽排序 | 当多个技能相关时决定调用顺序 |

**交互**:
- 技能列表带搜索过滤
- 每个技能显示：名称、描述、状态（启用/禁用）
- 支持「全选」「清空」快捷操作

---

#### 4.3.3 🔧 工具配置

**功能**: 配置该 Agent 可调用的外部工具（MCP Tools / Native Tools）。

**配置项**:

| 字段 | 类型 | 说明 |
|------|------|------|
| **工具列表** | 树形多选 | 按类别分组展示可用工具 |
| **工具参数** | 动态表单 | 选中工具后展示其必填/可选参数 |
| **工具开关** | Toggle | 快速启用/禁用单个工具 |
| **工具调用限制** | 数字输入 | 单次任务中该工具最多调用次数 |

**工具分类**:
```
📁 文件操作
  ├─ read (读取文件)
  ├─ write (写入文件)
  └─ exec (执行命令)

🌐 Web 能力
  ├─ web_search (搜索)
  └─ web_fetch (抓取页面)

💬 通讯
  ├─ feishu_doc (飞书文档)
  ├─ feishu_message (飞书消息)
  └─ email (邮件)

🔧 系统
  ├─ code_interpreter (代码执行)
  └─ image_analysis (图像分析)
```

**交互**:
- 树形结构支持展开/折叠
- 工具名称旁显示简要描述 Tooltip
- 批量操作支持「启用全部 Web 工具」等快捷操作

---

#### 4.3.4 💬 提示词编辑

**功能**: 编辑该 Agent 的系统提示词（System Prompt）和用户提示词模板。

**配置项**:

| 字段 | 类型 | 说明 |
|------|------|------|
| **角色定义 (Role)** | 文本输入 | 简短描述 Agent 身份，如"你是一个资深金融分析师" |
| **系统提示词** | 富文本编辑器 | System Prompt，支持 Markdown 语法 |
| **用户提示词模板** | 模板编辑器 | 支持变量占位符 `${variable}` |
| **输出格式约束** | 下拉多选 | JSON / Markdown / Plain Text / 自定义格式 |
| **对话示例** | 示例对话框 | 提供 Few-shot 示例，改进输出稳定性 |

**高级配置**:

| 字段 | 类型 | 说明 |
|------|------|------|
| **指令约束** | 多行文本 | 明确该 Agent "应当"和"不应当"做什么 |
| **上下文长度** | 数字输入 | 最大保留历史消息轮数 |
| **思维链 (CoT) 开关** | Toggle | 启用逐步推理模式 |

**交互**:
- 提供「AI 优化提示词」按钮，自动润色和改进
- 支持导入/导出提示词为 `.md` 文件
- 实时预览渲染后的 Markdown 效果

### 5.3 任务编辑器

#### 5.3.1 执行类型选择

创建或编辑任务时，首先选择执行类型：

| 执行类型 | 图标 | 描述 |
|----------|------|------|
| **顺序执行 (Sequential)** | ➡️ | 按步骤顺序执行，上一步完成才执行下一步 |
| **并行执行 (Parallel)** | ⚡ | 所有步骤同时执行，最后汇总结果 |
| **条件执行 (Conditional)** | 🔀 | 根据前置步骤结果决定后续执行路径 |

## 六、Tab 4: 协商协议配置 🆕

### 6.1 设计目标

为团队中的 Agent 通信选择合适的交互协议，并配置相关参数。

### 6.2 协议类型选择

**三种协议模式**:

| 协议 | 描述 | 适用场景 |
|------|------|----------|
| **File-based** 📁 | Agent 间通过读写共享文件传递信息 | 离线批处理、文件系统可用的环境 |
| **Protocol-based** 📡 | Agent 间通过结构化协议（如 JSON-RPC、GraphQL）直接通信 | 实时性要求高、需要精确消息格式 |
| **Hybrid** 🔀 | 混合模式，小消息走协议，大数据走文件 | 复杂任务、需要平衡实时性与大数据传输 |

### 6.3 协议选择 UI

## 八、数据模型

### 8.1 核心数据结构

```typescript
interface MultiAgentTeam {
  id: string;
  name: string;
  agents: Agent[];
  tasks: Task[];
  protocol: ProtocolConfig;
  createdAt: string;
  updatedAt: string;
}

interface Agent {
  id: string;
  name: string;
  description: string;
  llmConfig: LLMConfig;
  skills: Skill[];
  tools: Tool[];
  systemPrompt: SystemPrompt;
}

interface LLMConfig {
  model: string;
  temperature: number;
  maxTokens: number;
  topP: number;
  frequencyPenalty: number;
  presencePenalty: number;
  apiBaseUrl?: string;
  apiKey?: string;
}

interface Task {
  id: string;
  name: string;
  executionType: 'sequential' | 'parallel' | 'conditional';
  steps: Step[];
  createdAt: string;
}

interface Step {
  id: string;
  name: string;
  agentId: string;
  inputSource: 'upstream' | 'manual' | 'fixed';
  inputMapping: Record<string, string>;
  timeout: number;
  retryCount: number;
  outputVariable: string;
  // 条件执行专用
  condition?: string;
  conditionTrue?: Step[];
  conditionFalse?: Step[];
}

interface ProtocolConfig {
  mode: 'file' | 'protocol' | 'hybrid';
  fileConfig?: FileConfig;
  protocolConfig?: ProtocolSettings;
  hybridConfig?: HybridConfig;
  agentOverrides?: Record<string, ProtocolConfig>;
}
```

## 十、附录

### 10.1 术语表

| 术语 | 定义 |
|------|------|
| **Skill** | 可复用的能力模块，如"搜索"、"文档读取" |
| **Tool** | 原子级工具函数，如 `web_search`、`read file` |
| **System Prompt** | 系统级提示词，定义 Agent 的角色和行为约束 |
| **Execution Type** | 任务执行类型：顺序、并行、条件 |
| **Protocol Mode** | Agent 间通信协议：File-based / Protocol-based / Hybrid |
| **Condition Branch** | 条件分支，根据表达式结果选择执行路径 |

### 10.2 外部依赖

- 现有的 Skill 注册与管理系统
- 现有的 Tool MCP 集成
- LLM 模型供应商的 API 支持
- 文件系统（File-based 模式）
