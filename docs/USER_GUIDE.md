## 快速开始

### 1. 创建团队

1. 进入 **"团队配置"** Tab
2. 点击 **"新建团队"** 按钮
3. 输入团队名称（如：论文研究团队）
4. 点击 **"创建"**

### 2. 添加Agent

1. 在团队配置中找到刚创建的团队
2. 点击 **"添加Agent"**
3. 选择Agent角色（协调者/搜索专家/分析师/撰写专家/审核专家）

### 3. 配置Agent能力

1. 点击Agent卡片上的 **"配置"** 按钮
2. 在弹出的Modal中配置：
   - **LLM**: 选择Provider和模型
   - **技能**: 选择需要的技能
   - **工具**: 选择需要的工具
   - **提示词**: 编写System Prompt
3. 点击 **"保存"**

### 4. 创建工作流

1. 进入 **"任务编排"** Tab
2. 点击 **"创建工作流"**
3. 选择执行类型：
   - **顺序执行**: 按步骤依次执行
   - **并行执行**: 多步骤同时执行
   - **条件执行**: 根据条件选择分支
4. 添加步骤并选择每个步骤的Agent
5. 点击 **"保存"**

### 5. 执行工作流

1. 在 **"任务列表"** 或 **"任务监控"** Tab中找到任务
2. 点击 **"执行"** 按钮
3. 观察执行进度和日志

## 团队配置

### 新建团队

```
团队名称: [输入名称]
团队描述: [输入描述]
```

### 管理团队成员

- **查看**: 点击团队卡片查看详情
- **编辑**: 点击编辑按钮修改团队信息
- **删除**: 点击删除按钮移除团队

### 添加Agent到团队

1. 点击团队的 **"添加成员"**
2. 从预设角色中选择：
   - 🎯 **协调者 (Coordinator)**: 协调工作流程
   - 🔍 **搜索专家 (Searcher)**: 搜索和检索信息
   - 📊 **分析师 (Analyzer)**: 分析和处理数据
   - ✍️ **撰写专家 (Writer)**: 生成文本内容
   - ✅ **审核专家 (Reviewer)**: 审核和质量控制

## 工作流编排

### 执行类型

#### 顺序执行

```
[步骤1: 搜索] → [步骤2: 分析] → [步骤3: 撰写] → [步骤4: 审核]
```

#### 并行执行

```
         ┌─ [子任务A] ─┐
[开始] ──┼─ [子任务B] ─┼─ [汇总]
         └─ [子任务C] ─┘
```

#### 条件执行

```
[判断条件] ─┬─ [条件满足: 步骤A] ─┬─ [结束]
            └─ [条件不满足: 步骤B] ┘
```

### 创建工作流步骤

1. 点击 **"添加步骤"**
2. 填写步骤信息：
   - 步骤名称
   - 选择执行Agent
   - 输入/输出映射（可选）
3. 使用上下箭头调整步骤顺序

## 执行与监控

### 任务状态

| 状态 | 说明 |
|------|------|
| `pending` | 待执行 |
| `running` | 执行中 |
| `completed` | 已完成 |
| `failed` | 失败 |
| `blocked` | 阻塞 |

### 6步执行追踪

工作流执行时会显示6个标准步骤的进度：

```
1. [✓] 任务下发   - 任务已分配
2. [✓] 信息收集   - 收集必要数据
3. [✓] 数据分析   - 处理和分析
4. [✓] 计划制定   - 制定执行方案
5. [○] 任务执行   - 执行具体操作
6. [ ] 结果汇报   - 汇总和输出
```

### 查看执行结果

1. 在任务列表中点击 **"查看详情"**
2. 在 **"结果详情抽屉"** 中查看：
   - 执行概览
   - 各步骤状态
   - 推理链Timeline
   - 输出JSON

### 导出执行报告

点击任务详情中的 **"导出报告"** 按钮，可导出JSON格式的执行报告。

## 常见问题

### 保存失败怎么办？

1. **检查网络连接** - 确保浏览器可以访问后端API
2. **检查必填字段** - 确保填写了所有必填项（如团队名称）
3. **查看浏览器控制台** - 按F12打开Console查看具体错误
4. **刷新页面重试** - 有时候是临时网络问题

### Agent不执行任务？

1. 检查Agent是否已启用（卡片开关是ON状态）
2. 检查Agent的LLM配置是否正确
3. 检查工作流步骤是否正确分配了Agent

### 任务执行卡住？

1. 查看执行日志了解卡住的位置
2. 检查前置步骤是否正确完成
3. 尝试重新执行任务

### 如何调试？

1. 打开浏览器开发者工具（F12）
2. 切换到 **Network** 标签查看API请求
3. 切换到 **Console** 标签查看错误日志
