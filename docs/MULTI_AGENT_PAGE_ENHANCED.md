# MultiAgentPage 增强版设计文档

> **角色**: Andy（产品经理）  
> **日期**: 2026-04-02  
> **版本**: v3.0（增强版）  
> **核心定位**: 支持子Agent能力配置、工作流可视化编排、主Agent动态生成临时子Agent

---

## 一、设计核心理念

### 1.1 三大核心能力

| 能力 | 描述 | 用户价值 |
|------|------|----------|
| **调整多智能体** | 点击Agent卡片弹出配置Modal，深度定制LLM/技能/工具/提示词 | 精细化控制每个Agent的专业能力 |
| **编排多智能体** | 可视化拖拽工作流编排，支持顺序/并行/条件分支 | 零代码设计复杂协作流程 |
| **主Agent生成子Agent** | 主Agent运行时按需动态创建临时子Agent | 智能化资源调配，适应复杂任务 |

### 1.2 与v2.0的差异

| 功能 | v2.0 | v3.0（增强版） |
|------|------|----------------|
| Agent配置 | Tab页面内配置 | **点击卡片弹出Modal**，更直观 |
| 工作流编排 | 列表式编辑 | **可视化拖拽编排**，所见即所得 |
| 子Agent生成 | 不支持 | **主Agent运行时动态生成**，按需创建 |
| 临时Agent管理 | 无 | **自动销毁机制**，资源清理 |

### 2.2 Agent卡片设计

```
┌─────────────────────────────────────┐
│  🤖 Agent-A                    [⋮]  │  ← 更多操作菜单
│  数据采集专家                        │
├─────────────────────────────────────┤

└─────────────────────────────────────┘
```

**交互**:
- 点击卡片空白区域 → 选中该Agent
- 点击 **[⚙️ 配置]** → 弹出 **Agent配置Modal**
- 点击 **[👁 预览]** → 弹出快速预览Panel
- 拖拽卡片 → 调整在列表中的顺序

| **结束节点** | 🏁 | 工作流终点，可有多个 |
| **步骤节点** | 📋 | 执行单个Agent任务 |
| **条件节点** | 🔀 | 根据条件分支 |
| **并行入口** | ⚡ | 并行执行开始 |
| **并行汇合** | 🔗 | 并行执行汇合 |
| **子工作流** | 📦 | 嵌套子工作流 |

### 4.3 拖拽操作

```
创建步骤:
1. 从工具栏拖拽 [📋 步骤] 到画布
2. 弹出步骤配置侧边栏
3. 选择执行Agent
4. 配置输入/输出
5. 连接流程线

创建条件分支:
1. 从工具栏拖拽 [🔀 条件] 到画布
2. 条件节点自动分出2条线(是/否)
3. 点击线条设置条件表达式
4. 拖拽Agent到对应分支

并行执行:
1. 从工具栏拖拽 [⚡ 并行] 到画布
2. 选择并行组内的Agent
3. 设置汇合策略(全部完成/任一完成)

调整顺序:
- 拖拽节点移动位置
- 点击连接线重新连接
- 右键节点删除
```

## 五、主Agent动态生成子Agent 🆕

### 5.1 核心概念

**主Agent生成子Agent** 是指在主Agent运行时，根据任务需求智能判断是否需要创建临时子Agent来完成特定子任务。

```
┌─────────────────────────────────────────────────────────────────┐
│                     主Agent运行时生成子Agent流程                  │
└─────────────────────────────────────────────────────────────────┘

    主Agent接受任务
           │
           ▼
    ┌──────────────┐
    │ 复杂度评估    │ ← 主Agent分析任务复杂度
    └──────┬───────┘
           │
           ▼
    ┌──────────────┐
    │ 复杂度 > 阈值? │ ──否──→ 直接执行
    └──────┬───────┘
           │是
           ▼
    ┌──────────────┐
    │ 是否有合适    │ ──否──→ 询问用户
    │ 现有Agent?    │        是否需要生成
    └──────┬───────┘
           │是
           ▼
    ┌──────────────┐
    │ 动态生成      │ ← 根据主Agent的提示词
    │ 子Agent      │   智能推断子Agent能力
    └──────┬───────┘
           │
           ▼
    ┌──────────────┐
    │ 子Agent      │
    │ 执行子任务    │
    └──────┬───────┘
           │
           ▼
    ┌──────────────┐
    │ 结果汇总      │
    │ 子Agent销毁  │
    └──────────────┘
```

## 七、数据结构

### 7.1 核心接口定义

```typescript
// Agent配置
interface AgentConfig {
  id: string;
  name: string;
  description: string;
  avatar?: string;
  llm: LLMConfig;
  skills: string[];          // Skill ID列表
  tools: ToolConfig[];
  systemPrompt: string;
  temperature: number;
  maxTokens: number;
  // 子Agent生成配置
  subAgentConfig?: SubAgentConfig;
}

interface SubAgentConfig {
  enabled: boolean;
  strategy: 'strict' | 'smart' | 'hybrid';
  maxConcurrency: number;
  maxLifetime: 'single-turn' | 'task-complete' | 'manual';
  memoryLimit: number;
  triggerConditions: TriggerCondition[];
  capabilityInheritance: {
    skills: boolean;
    tools: boolean;
    llm: boolean;
    context: boolean;
    contextLimit: number;
  };
}

interface TriggerCondition {
  type: 'complexity' | 'skill-required' | 'user-request' | 'multi-step';
  threshold?: number;
  skillMatch?: string;
  keyword?: string;
}

// 工作流配置
interface Workflow {
  id: string;
  name: string;
  nodes: WorkflowNode[];
  edges: WorkflowEdge[];
  variables: Record<string, any>;
}

interface WorkflowNode {
  id: string;
  type: 'start' | 'end' | 'step' | 'condition' | 'parallel' | 'subflow';
  position: { x: number; y: number };
  data: StepConfig | ConditionConfig | ParallelConfig;
}

interface StepConfig {
  name: string;
  agentId: string;
  inputMapping: Record<string, string>;
  outputVariable: string;
  timeout: number;
  retryCount: number;
  failStrategy: 'stop' | 'continue' | 'fallback';
}

interface ConditionConfig {
  expression: string;
  trueBranch: string[];      // Node IDs
  falseBranch: string[];     // Node IDs
  elseBranch?: string[];     // Node IDs
}

// 运行时状态
interface RuntimeState {
  workflowId: string;
  status: 'pending' | 'running' | 'paused' | 'completed' | 'failed';
  currentNodeId: string;
  executedNodes: string[];
  activeSubAgents: SubAgent[];
  logs: ExecutionLog[];
}
```
